3. Міжнародна SEO-архітектура (hreflang)

Hreflang - це HTML-атрибут, який вказує пошуковим системам, яка мовна або регіональна версія сторінки призначена для якої аудиторії. Без коректного hreflang Google може показувати українську версію американцям або навпаки, що знижує CTR і конверсію.

3.1 Поточний стан hreflang

Реалізований через Yoast SEO. В <head> присутні:


<link rel="alternate" href="https://salesdep.ai/" hreflang="en" /><link rel="alternate" href="https://salesdep.ai/uk/" hreflang="uk" />


3.2 Виявлені проблеми

Проблема

Пояснення

Пріоритет

Відсутній hreflang="x-default"

Без x-default Google не знає, яку версію показувати при невизначеному гео (VPN, нова країна). Ризик: обидві версії конкурують за одних і тих самих користувачів.

КРИТИЧНО

hreflang="en" замість "en-US"

Для таргетингу конкретно США потрібна регіональна специфікація en-US. Тег en охоплює всіх англомовних користувачів у світі.

ВИСОКА

Один домен для двох ринків

Google Search Console може бути налаштований лише під один основний гео. Це не дозволяє паралельно таргетувати US і UA через один домен.

4.2 SoftwareApplication - базовий тип для SaaS

SoftwareApplication дозволяє Google відображати rich results з рейтингом, платформою та категорією додатку. Це прямо підвищує CTR у порівнянні зі звичайним сніппетом.

Рекомендована розмітка SoftwareApplication

{ "@context": "https://schema.org", "@type": "SoftwareApplication", "name": "Salesdep.ai", "applicationCategory": "BusinessApplication", "operatingSystem": "Web", "url": "https://salesdep.ai/", "description": "AI-powered sales automation for B2B teams. Automate outreach and qualify leads 24/7.", "offers": { "@type": "AggregateOffer", "priceCurrency": "USD", "lowPrice": "0", "highPrice": "499", "offerCount": "5" }, "aggregateRating": { "@type": "AggregateRating", "ratingValue": "4.7", "reviewCount": "24" }}

Примітка: aggregateRating має відповідати реальним відгукам (G2, Capterra тощо). Не можна вигадувати рейтинг.


4.3 FAQPage - розширений блок у SERP

На сайті salesdep.ai є блок з 4 питаннями у стилі FAQ. Розмітки FAQPage при цьому немає. FAQPage дозволяє Google розгорнути відповіді прямо у пошуковій видачі, що збільшує займану площу сніппету і підвищує CTR на 20–30%.

Рекомендована розмітка FAQPage

{ "@context": "https://schema.org", "@type": "FAQPage", "mainEntity": [ { "@type": "Question", "name": "What is Salesdep.ai?", "acceptedAnswer": { "@type": "Answer", "text": "Salesdep.ai is an AI sales automation platform that handles outreach, lead qualification and follow-ups 24/7." } }, { "@type": "Question", "name": "How does AI sales automation work?", "acceptedAnswer": { "@type": "Answer", "text": "The AI manager engages leads via text or voice, qualifies them and books meetings automatically." } } ]}

Розмітку слід додати у <script type="application/ld+json"> на головній сторінці або через Yoast SEO Custom Schema.


4.4 Offer - тарифні плани

Сайт має 5 тарифних планів: Free, Starter, Professional, Business, Enterprise. Без розмітки Offer ця інформація недоступна для структурованих результатів. При наявності розмітки Google може відображати ціни прямо у SERP.

7.3 Безпека

  • WordPress generator meta: приховати через functions.php або плагін безпеки, щоб не розкривати версію CMS для ботів.

Приклад: приховати generator у functions.php

// Додати у wp-content/themes/[your-theme]/functions.phpremove_action('wp_head', 'wp_generator');

  • xmlrpc.php: заблокувати через .htaccess або плагін Wordfence. XML-RPC відкриває вразливість до brute-force атак і DDoS через pingback.

Приклад: заблокувати xmlrpc.php через .htaccess

# Додати у .htaccess<Files xmlrpc.php> Order Deny,Allow Deny from all</Files>

  • RSS feed /sample-page/feed/: WordPress-артефакт від тестової сторінки. Видалити сторінку або замінити redirect.

  • Відкласти завантаження некритичного JS: додати defer або async до скриптів.

  • Використовувати CDN для статичних ресурсів (Cloudflare або BunnyCDN).

  • Зменшити TTFB: оптимізувати запити до бази даних WordPress, увімкнути кешування (WP Rocket або W3 Total Cache).
